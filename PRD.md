## 1. 실행 요약 (Executive Summary)

### 1.1 프로젝트 개요

**Pixel Agent Desk**는 Claude CLI 사용자가 AI 에이전트들의 활동을 시각적으로 확인하고 관리할 수 있는 데스크톱 애플리케이션입니다. 현재 터미널 환경의 제약을 넘어, 직관적인 시각화로 AI 활용 개발의 생산성과 즐거움을 극대화하는 것을 목표로 합니다.

**현재 상태 (2026-03-05 기준):**
- **기술적 완성도**: 90% (핵심 기능 구현 완료)
- **테스트 커버리지**: 0% (기술 부채로 긴급 개선 필요)
- **운영체제**: Windows 전용 (macOS/Linux 지원은 6개월 후 검토)
- **사용자 기반**: 오픈소스 커뮤니티 초기 단계

**개발 현황:**
- 구현된 기능: Hook 기반 이벤트 수신, 상태 시각화, PID 기반 생명주기 관리, 터미널 포커싱, 자동 세션 복구, 멀티 에이전트 관리, 프로젝트 그룹화
- 긴급 수정 필요: 메모리 누수, 접근성 기본 요구사항, 단위 테스트 0% 커버리지

### 1.2 핵심 변경사항

#### 변경 1: 타겟 사용자 재정의
**이전:** CLI 파워 유저 개발자 (중립적)
**재정의:**
- **1차 타겟:** AI 도구 얼리 어답터 (20-35세, 시각적 피드백 선호)
  - 하루 2-4시간 Claude CLI 활용
  - 2-3개 프로젝트 동시 진행

**2순위: 즐거움 (Delight)**
- **정의**: 개발 작업에 재미와 성취감을 부여하는 감성적 가치
- **실현 방법**:

- **균형**: 귀여움과 전문성의 균형 (과도한 장난기 자제)
- **측정 가능한 지표**:
  - 재미 요인 NPS: "픽셀 아트 캐릭터가 즐겁습니다" 항목 60점 이상
  - 애니메이션 상호작용률: 일일 평균 10회 이상 캐릭터 클릭/호버
  - 커스터마이징 사용률: 30% 이상 사용자가 아바타/테마 변경

**3순위: 통찰 (Insight)**

- **실현 방법**:
  - 시간 추적으로 작업 시간 자동 측정
  - 상태 전이 빈도로 작업 패턴 분석
  - 팀원 작업 현황 투명하게 공유
- **측정 가능한 지표**:
  - 터미널 전환 횟수 감소율: 50% 이상 (베이스라인 대비)
  - 작업 시간 파악 정확도: 90% 이상
  - 생산성 향상감: 70% 이상 사용자가 생산성 향상 체감

**4순위: 신뢰성 (Reliability)**
- **정의**: 기술적 안정성으로 사용자가 믿고 사용할 수 있는 기반
- **실현 방법**:
  - 99.9% 업타임 보장
  - <100ms 상태 전환으로 즉각적 피드백
  - 자동 세션 복구로 데이터 손실 방지
- **측정 가능한 지표**:
  - 앱 업타임: 99.9% (월간)
  - 충돌율: < 0.1% (P0 충돌 0%)
  - 세션 복구 성공률: 95% 이상

#### 아키텍처 원칙

**1. 관심사 분리 (Separation of Concerns)**
- 데이터 수집, 비즈니스 로직, 프레젠테이션 계층 분리
- 각 컴포넌트의 단일 책임 명확히 정의

**2. 이벤트 기반 아키텍처 (Event-Driven Architecture)**
- Observer 패턴을 통한 느슨한 결합
- EventEmitter를 통한 비동기 통신

**3. 의존성 주입 (Dependency Injection)**
- 구체적 구현이 아닌 추상화에 의존
- 테스트 용이성 및 확장성 확보
- Inversion of Control (IoC) 컨테이너 고려

**4. 실패 허용 설계 (Fail-Silent Design)**
- Hook 실패 시 Claude CLI 영향 최소화
- 에러 발생 시에도 앱 안정성 유지

## 부록

### A. 용어 정리

- **Hook**: Claude CLI의 이벤트 시스템으로, 특정 이벤트 발생 시 외부 프로그램 실행
- **PID**: Process Identifier, 운영체제가 프로세스를 식별하는 고유 번호
- **WMI**: Windows Management Instrumentation, Windows 시스템 관리를 위한 인터페이스
- **IPC**: Inter-Process Communication, 프로세스 간 통신 메커니즘
- **WCAG**: Web Content Accessibility Guidelines, 웹 콘텐츠 접근성 가이드라인
- **MoSCoW**: Must Have, Should Have, Could Have, Won't Have 우선순위 방법론
- **PMF**: Product-Market Fit, 제품-시장 적합성
- **NPS**: Net Promoter Score, 추천 의향 지표
- **CSAT**: Customer Satisfaction, 고객 만족도
- **MAU**: Monthly Active Users, 월간 활성 사용자
- **DAU**: Daily Active Users, 일간 활성 사용자
